### Suggesting Enhancements

This section guides you through submitting an enhancement suggestion for CSP Sudoku Visualizer, including completely new features and minor improvements to existing functionality. Following these guidelines helps maintainers and the community understand your suggestion and find related suggestions.

### Your First Code Contribution

1.  **Fork the repository**
2.  **Clone your fork**:
    \\\ash
    git clone https://github.com/YOUR-USERNAME/csp-sudoku-visualizer.git
    \\\
3.  **Create a branch**:
    \\\ash
    git checkout -b feature/AmazingFeature
    \\\
4.  **Install dependencies**:
    \\\ash
    npm install
    \\\
5.  **Make your changes** and commit them using [Conventional Commits](https://www.conventionalcommits.org/):
    \\\ash
    git commit -m "feat: add new degree heuristic"
    \\\
6.  **Push to your fork**:
    \\\ash
    git push origin feature/AmazingFeature
    \\\
7.  **Open a Pull Request** targeting the \main\ branch.

## Style Guide

*   **Linting**: Run \
pm run lint\ before committing to ensure code style consistency.
*   **React**: Functional components with Hooks are preferred over Class components.
*   **Algorithms**: Keep core logic pure (in \src/algorithms/\) and separate from UI components.
